# Socket io

## what are web sockets ?

Bidirectional and low-latency communication for every platform

- Bidirectional: the connection can go both ways between server(the server can sent messages out to who ever needs it ) and clients over TCP.
- Low-latency: there is almost now delay, small amount of time required to send messages out (real-time)
- Every platform: it allows different devices & softwares to use the same communication style on real-time over TCP.

  **socket.io** is library built on the top of web Socket API so the client (js or browsers) can use real-time communication

  **Long polling** is a technique used to simulate real-time communication over standard HTTP before technologies like WebSocket became common. It works by the client sending a request to the server, which then keeps the connection open until new data is available instead of responding immediately; once the server sends the data, the client instantly makes another request, creating a continuous loop. This approach allows applications like chats or notifications to feel “live,” but it’s less efficient than modern solutions because it repeatedly opens new connections rather than maintaining a single persistent one.
  **Multiplexing**: means sending multiple types of data or messages over a single connection at the same time by organizing them into separate logical channels or rooms.
  **Re-connecting:** In WebSocket, reconnecting is the process of automatically re-establishing the connection when it drops to keep communication continuous.

  **Buffering:**buffering is a method of temporarily storing messages when data is being sent faster than it can be processed to prevent overload or data loss.
